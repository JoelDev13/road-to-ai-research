#### Console programs

- A console program is a box: it takes **input** (text) and returns **output** (text).
- Base structure of every Python program:

```python
  def main():
      # your code here

  if __name__ == "__main__":
      main()
```

  The `if __name__ == "__main__":` line is what "triggers" main(), equivalent to Karel's implicit run()

- **Output**: `print("text")`. The quotes don't show up in the console, they just mark where the string starts/ends

- **Input**: `input("prompt: ")` does 2 things: (1) prints the prompt, (2) pauses the program until the user types something and hits Enter

- **`input()` ALWAYS returns a string**, even if the user types numbers. That's why you need to cast:

```python
  num1 = input("Enter number: ")  # "19" (string)
  num1 = int(num1)                # 19 (int)
```

- To display output combining text + variables, you can concatenate with `+` (converting everything to string with `str()`) or use **f-strings**: `print(f"The total is {total}")`

#### Variables

- A variable is a suitcase in RAM: it has a **name** (tag) and a **value** (contents)

- `x = 5` creates the variable if it doesn't exist, or updates its value if it already does

- **Naming rules**:
  - Must start with a letter or `_`
  - Can only contain letters, digits, `_`
  - Cannot be a reserved keyword (`def`, `for`, etc.)
  - Are *case-sensitive* (`Hello` â‰  `hello`)
  - Should be descriptive and written in `snake_case`

- **Data types**:
  | Type | Example |
  |---|---|
  | `int` | `num_students = 200` |
  | `float` | `sales_tax = 8.1` |
  | `string` | `my_name = "Neel"` |
  | `bool` | `month_is_june = True` |
- Python is **dynamically typed**: you don't declare the type, it's inferred automatically (unlike C, where you'd write `int num_students = 5;`)

- **Constants**: convention is to name them in uppercase `SNAKE_CASE` (`INCHES_IN_FOOT = 12`) to signal they shouldn't change. It's just a convention, Python doesn't enforce it

#### Expressions

- **Arithmetic operators** (with num1=5, num2=2):
  | Operator | Name | Result |
  |---|---|---|
  | `+` | addition | `7` |
  | `-` | subtraction | `3` |
  | `*` | multiplication | `10` |
  | `/` | true division | `2.5` (always float) |
  | `//` | integer (floor) division | `2` (truncates the decimal) |
  | `%` | modulo/remainder | `1` |
  | `**` | exponentiation | `25` |
  | `-x` | negation | `-10` (num1 * -num2) |

- **Precedence**: follows PEMDAS, left to right. Ex: `1 + 3*5/2 = 1 + 7.5 = 8.5`

- **Type conversion**:
  - *Implicit*: int op int â†’ int (except `/`, which always gives a float). If you mix int and float â†’ result is float.
  - *Explicit*: `float()` and `int()`. Note: `int()` truncates, it doesn't round

- **Expression shorthands** (compound assignment):
```python
  num1 += 1   # equivalent to num1 = num1 + 1
  num1 -= 1
  num1 *= 2
  num1 /= 2
```

  If you just write `num1 + 2` without assigning it to anything, Python evaluates the expression but the result is lost (not printed, not stored)

- **`math` library** (requires `import math`):
  | Function | Does |
  |---|---|
  | `math.e`, `math.pi` | constants |
  | `math.sqrt(x)` | square root |
  | `math.exp(x)` | e raised to x |
  | `math.log(x)` | natural logarithm |

#### Randomness

- There's no "true" randomness on a computer â€” it's **pseudorandomness**, generated by algorithms.
- Requires `import random`.
- **`random` module functions**:

  | Function | Returns |
  |---|---|
  | `random.randint(min, max)` | random integer between min and max (**both inclusive**) |
  | `random.random()` | random float between 0 and 1 |
  | `random.uniform(min, max)` | random float between min and max |
  
- Usage pattern (not the full solution): declare a constant like `NUM_SIDES = 6`, and use `random.randint(1, NUM_SIDES)` to simulate rolling a die, storing the result in a variable
